Disclaimer… not an expert at CKAN.

According to their installation documentation it sounds like I should install CKAN from source since it is the most customization vs package installation and should have a easier deployment process vs Docker Compose (source install deployment).

CKAN runs on Linux and since we will be installing from source and not via docker. It’s good to invest some time in setting a efficient Linux environment instead of working all day on command line: Ubuntu Desktop/Virtual Box, PyCharm as a python IDE and VS Code for front end development, GitHub Desktop for version control. Also since CKAN is built on Django/python (I think???) add a Django/Python IDE (going to try PyCharm).

Note in Linux,there are two popular package managers (think of CRAN): SNAP and apt. Most software have installation instructions on their get started page.
